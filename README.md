# Moment Maker

A playful mobile app that kills "what should we do today?" decision fatigue. Flick through head-to-head idea cards, watch your plan brew, and open the gift to reveal a personalized experience plan.

See [CLAUDE.md](./CLAUDE.md) for the full product spec, design language, and build order.

## Status

**M1 — core loop skeleton** ✅
**M2 — physics & sound** ✅
**M3 — the algorithm** ✅
**M4 — the rest of the app** ✅
**M5 — juice pass** ✅

- Expo (SDK 57) + expo-router + TypeScript strict
- 130 seed ideas generated into `assets/seedIdeas.json` and loaded into SQLite on first launch
- Home screen: who's playing / time / budget → "Deal me in", global sound toggle
- Picker screen: cards are **physically throwable** (velocity or ~120pt displacement locks in, gentle release springs back), velocity-scaled pentatonic marimba ticks while dragging, woodblock pickup tap, two-note throw lock-in, deal-in plinks, haptics scaled to throw strength
- Gift reveal: breathing box, 0.4s shake with woodblock knocks, lid-pop + confetti burst, plan unfolds on an ascending marimba melody resolving into a chord
- Reduce-motion: tap-to-pick fallback, no throws/shake/confetti
- Sound palette generated by `node scripts/generate-sounds.mjs` (placeholder additive-synth marimba/woodblock — swap for recorded samples in M5)
- **The algorithm** (`src/lib/algorithm/`, pure TS, unit-tested — `npm test`): content-based preference model that learns on every pick (velocity-scaled learning rate), hard filter → scoring with cold-start crowd-pleaser blend + novelty penalties, informative pairing with 12% exploration wildcards, complementary plan assembly under the time budget
- Onboarding quiz (6 contrasting throws) seeds the profile on first launch; a "How was it?" card on home feeds 1–5★ ratings back at 3× strength
- **Plan view**: swap any step (re-rolled by the algorithm), "find nearby" map links on travel steps, "We did it! 🎉" completion with confetti + inline rating
- **Scrapbook**: past plans as keepsake cards — date, steps, tappable stars, photo attachments (copied into app storage) on completed plans
- **Browse**: the whole idea deck with filters (mood, budget, duration, group, indoor/outdoor, walking-distance, energy, time of day) and weather-aware badges
- **Add your own idea**: friendly chip form; the idea joins the deck and teaches the profile at 2× strength
- **Weather**: coarse location + Open-Meteo (no key), 3-hour cache, feeds the session hard filter and the "72° clear" chips; silently degrades to "unknown" without permission
- **Settings**: sound/music/haptics toggles, location permission, reset-profile (scrapbook survives)
- **Time-of-day theming**: the canvas warms and cools with the clock (morning peach → midday cream → dusk orange → night lavender-blue), synced with the music
- **Adaptive music**: four gentle placeholder loops (morning/day/dusk/night), all in the C-pentatonic family so drag ticks are always in key; quiet, looped seamlessly, behind the global mute and its own toggle
- Micro-animations: breathing hero button, staggered scrapbook shelf, all gated by reduce-motion

## Run it

```bash
npm install
npx expo start
```

Scan the QR code with **Expo Go** on a real device — the card feel can't be judged in a simulator.

## Structure

- `src/app/` — expo-router screens (home → picker → reveal)
- `src/components/` — IdeaCard, VsBadge, BrewMeter, chips & buttons
- `src/lib/db/` — expo-sqlite schema + seed loading
- `src/lib/store/` — zustand session state (pairs, picks, plan assembly)
- `assets/seedIdeas.json` — the seed idea database
