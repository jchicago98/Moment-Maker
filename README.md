# Moment Maker

A playful mobile app that kills "what should we do today?" decision fatigue. Flick through head-to-head idea cards, watch your plan brew, and open the gift to reveal a personalized experience plan.

See [CLAUDE.md](./CLAUDE.md) for the full product spec, design language, and build order.

## Status

**M1 — core loop skeleton** ✅
**M2 — physics & sound** ✅

- Expo (SDK 57) + expo-router + TypeScript strict
- 130 seed ideas generated into `assets/seedIdeas.json` and loaded into SQLite on first launch
- Home screen: who's playing / time / budget → "Deal me in", global sound toggle
- Picker screen: cards are **physically throwable** (velocity or ~120pt displacement locks in, gentle release springs back), velocity-scaled pentatonic marimba ticks while dragging, woodblock pickup tap, two-note throw lock-in, deal-in plinks, haptics scaled to throw strength
- Gift reveal: breathing box, 0.4s shake with woodblock knocks, lid-pop + confetti burst, plan unfolds on an ascending marimba melody resolving into a chord
- Reduce-motion: tap-to-pick fallback, no throws/shake/confetti
- Sound palette generated by `node scripts/generate-sounds.mjs` (placeholder additive-synth marimba/woodblock — swap for recorded samples in M5)

Coming next (M3): the learning algorithm — profile weights, learning on every pick, scoring/pairing, real plan assembly, onboarding quiz.

## Run it

```bash
npm install
npx expo start
```

Scan the QR code with **Expo Go** on a real device — the card feel can't be judged in a simulator.

## Structure

- `src/app/` — expo-router screens (home → picker → reveal)
- `src/components/` — IdeaCard, VsBadge, BrewMeter, chips & buttons
- `src/lib/db/` — expo-sqlite schema + seed loading
- `src/lib/store/` — zustand session state (pairs, picks, plan assembly)
- `assets/seedIdeas.json` — the seed idea database
